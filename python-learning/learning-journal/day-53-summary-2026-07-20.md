# Day 53 学习总结：多文档小样本评测

今天把 `/evaluation` 从固定问题评测扩展成了支持自定义 evaluation cases 的评测接口。

## 今天学到的重点

评测不能只绑定一份文档。

更合理的结构是：

```text
文档 A -> 问题集 A
文档 B -> 问题集 B
```

因为不同文档的主题、chunk 编号和期望答案都不同。

## 当前变化

现在 `/evaluation` 可以：

```text
不传 evaluation_cases -> 使用默认 10 题 FAQ
传 evaluation_cases -> 使用自定义问题集
```

## 项目表达

现在可以说：

```text
设计了可配置 evaluation cases 的检索评测接口，支持对不同文档配置不同问题集，并用 Top-1 hit / Top-k recall 量化检索效果。
```
