## 今日关键发现

### 1. 新版 Dify 使用 Chatflow

原计划里写的是：

```text
Chatbot -> Context
```

但实际页面没有 Chatbot 选项，因此改为：

```text
Chatflow -> Knowledge Retrieval
```

这说明以后不能死记页面名称，要抓住本质：

```text
把知识库检索结果送给 LLM
```

### 2. 倒排索引对字面匹配敏感

第一次问：

```text
Day 8 学了什么？
```

检索失败，可能和 `Day 8` / `Day8` 的空格差异有关。

当前知识库检索使用倒排索引，关键词匹配更依赖字面形式。

### 3. Hybrid Search 需要 embedding 模型

如果想使用 Hybrid Search，需要 embedding 模型支持。

DeepSeek 当前继续作为生成模型使用，但 embedding 模型需要后续单独选择。可选方向包括：

- Dify Cloud 可用的 embedding provider。
- 本地 BGE embedding。
- Jina Embeddings。
- 其他低成本 embedding API。
- 以后如果 OpenAI API 计费解决，也可以使用 OpenAI embedding。

### 4. 复制调试内容时要区分最终回答

排查过程中复制过调试/思考内容，但这不等于最终用户回答一定会展示这些内容。

记录 RAG 测试结果时，需要区分：

- 检索节点或调试面板中的中间信息。
- 用户最终看到的 Answer 输出。
