# Day 38 学习总结：抽离 Chunking Service

## 今天完成了什么

今天把文档切分逻辑从 Streamlit `app.py` 中抽离到了：

```text
backend/services/chunking.py
```

现在 Streamlit 页面会导入并复用后端 service。

## 为什么这样做

v2 的目标是把项目从 Streamlit 单体原型升级成更工程化的系统。

第一步不是重写所有代码，而是逐步拆出边界清楚的模块。

chunking 很适合作为第一个模块，因为它：

- 输入输出清楚。
- 不依赖模型。
- 不依赖 Chroma。
- 不依赖 API Key。
- 容易验证。

## 验证结果

FAQ 文档仍然切出：

```text
6 chunks
```

说明抽离没有破坏现有切分行为。

## 下一步

下一步建议抽离 embedding service，把 Teaching keyword embedding 和 BGE Chinese embedding 统一管理起来。
