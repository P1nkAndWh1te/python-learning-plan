# Day 52 学习总结：FastAPI Multipart 文件上传

今天给 DocuAsk 后端增加了真正的文件上传入口。

## 今天学到的重点

同一个业务能力可以有多个入口：

```text
/documents：JSON text 入库
/documents/upload：multipart 文件入库
```

但入口变多不代表逻辑要复制。

更好的做法是：

```text
不同入口 -> 统一转换成 text -> 复用同一个入库函数
```

## 当前后端链路

现在后端已经支持：

```text
POST /documents
POST /documents/upload
POST /qa
POST /answer
POST /evaluation
```

## 最重要的认知

文件上传不是 RAG 的核心算法，但它让系统从“脚本演示”更接近真实应用。

简历或面试里可以表达为：

```text
补充 multipart 文件上传接口，打通从本地文档上传到 Chroma 入库的后端链路。
```
