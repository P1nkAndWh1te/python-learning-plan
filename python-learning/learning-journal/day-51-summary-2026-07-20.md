# Day 51 学习总结：FastAPI 接入 LLM Answer

今天把 LLM 生成能力从 Streamlit 页面侧抽成了可复用 service，并接入 FastAPI `/answer`。

## 今天学到的重点

RAG 后端可以分成两层：

```text
/qa：检索层
/answer：生成层
```

这样排查问题更清楚：

- 如果 `/qa` 检索错了，问题在 embedding / vector db / BM25 / RRF。
- 如果 `/qa` 检索对了但 `/answer` 回答错了，问题更可能在 prompt 或 LLM 生成。

## 为什么测试不直接调用 DeepSeek

因为真实 LLM API 依赖：

- API Key。
- 网络。
- 额度。
- 模型服务状态。

这些都不是稳定的本地测试条件。

所以 pytest 只验证：

```text
没有 DEEPSEEK_API_KEY 时，/answer 返回 503
```

真实生成可以用 Day51 验证脚本在有 key 的环境下手动测试。

## 当前项目表达

现在可以更准确地说：

```text
后端已支持文档入库、检索问答、检索评测和基于检索上下文的 LLM answer 生成。
```
