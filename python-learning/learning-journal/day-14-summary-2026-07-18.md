## 当前能说清楚的 RAG 链路

```text
原始文档
  -> 文档解析
  -> 分块
  -> embedding
  -> 存入向量数据库
  -> 用户提问
  -> 检索相关片段
  -> LLM 基于片段生成回答
  -> 返回回答和引用来源
```

---

## 当前技术结论

- DeepSeek 继续作为生成模型。
- embedding 模型后续单独选择。
- Chroma 作为向量数据库主线。
- LlamaIndex 作为 RAG 框架主线。
- Streamlit 作为项目 UI。
- SQL 用于结构化数据查询，不替代向量检索。
