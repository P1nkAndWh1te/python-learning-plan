## 今日完成情况

状态：完成 DeepSeek 生成回答的代码接入。

今天在 Day19 的检索上下文基础上，加入了 DeepSeek 生成步骤。现在应用会先检索相关 chunk，再把这些 chunk 格式化为 prompt 上下文，最后调用 DeepSeek 生成最终回答。

---

## 已实现功能

- 从环境变量读取 `DEEPSEEK_API_KEY`。
- 使用 OpenAI-compatible SDK 连接 DeepSeek。
- 使用 `https://api.deepseek.com` 作为 base URL。
- 使用 `deepseek-v4-flash` 作为生成模型。
- 构造 RAG prompt，限制模型只能根据检索资料回答。
- 要求最终回答保留来源引用。
- 对缺少 API Key、额度不足和 API 请求失败做页面提示。

---

## 当前边界

当前应用已经完成：

```text
文档上传
-> 文档读取
-> 文档切分
-> 教学版 embedding
-> Chroma 检索
-> 上下文和来源引用
-> DeepSeek 生成回答
```

仍然存在的限制：

- 当前 embedding 是教学版关键词向量，不是真实语义 embedding。
- 暂不支持 PDF。
- Chroma 当前使用临时 collection，没有持久化知识库。
- 当前 Codex 执行环境没有设置 `DEEPSEEK_API_KEY`，因此本地只验证了缺少 key 的提示路径，没有完成真实 API 响应验证。
