## 今日完成情况

状态：完成教学版 embedding 和 Chroma 检索。

今天在 Day17 的 chunk 切分基础上，加入了一个最小可用的检索链路。上传文档后，程序会把 chunk 转成向量，写入 Chroma 临时 collection，然后根据用户问题检索相关 chunk。

---

## 已实现功能

- 定义关键词维度作为教学版 embedding。
- 将每个 chunk 转成关键词计数向量。
- 将 chunk、embedding 和 metadata 写入 Chroma。
- 将用户问题转成同一套向量。
- 使用 Chroma 检索 Top 3 相关 chunk。
- 页面显示检索排名、chunk 编号、距离和内容。

---

## 当前边界

今天使用的是教学版 embedding，不是真实语义 embedding。

它只能根据关键词命中情况进行检索，因此适合学习链路，不适合当作最终项目效果。

当前应用已经完成：

```text
文档上传 -> 文档读取 -> 文档切分 -> 教学版 embedding -> Chroma 检索 -> 展示相关 chunk
```

还没有完成：

```text
真实 embedding -> 更稳定的检索 -> LLM 生成最终回答 -> 来源引用
```

---

## 今日理解

向量数据库不是直接“理解文档”的工具。

它做的是：

```text
存储文本片段
存储文本片段对应的向量
根据问题向量找相近的文本片段
```

embedding 的质量决定了检索质量。今天的关键词向量能帮助理解流程，但真实项目后续需要换成更好的 embedding 方案。
