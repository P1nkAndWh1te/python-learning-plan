## 今日 RAG 链路

今天的流程是：

```text
用户问题
  -> LlamaIndex 建立索引
  -> Retriever 检索相关文档片段
  -> 拼接 retrieved context
  -> DeepSeek API 根据 context 生成回答
```

对应职责：

- **LlamaIndex**：负责加载文档、建立索引、检索相关片段。
- **本地关键词 embedding**：临时负责把文本转成向量，用于检索。
- **DeepSeek**：负责根据检索到的资料生成最终回答。

## 今日踩坑

### 1. API Key 认证失败

第一次运行 Day13 时出现：

```text
401 authentication_error
Your api key is invalid
```

原因：当前 PowerShell 窗口里的 `DEEPSEEK_API_KEY` 可能是旧值或设置错误。

解决：

1. 重新设置当前窗口的 `DEEPSEEK_API_KEY`。
2. 先运行 Day8 最小调用脚本确认 Key 有效。
3. 再运行 Day13 RAG 脚本。

可复用经验：

```text
RAG 脚本调用失败时，先拆开验证：先测 API Key，再测检索，再测组合链路。
```

## 今日核心理解

### 检索和生成是两步

```text
检索：从知识库里找相关资料。
生成：让 LLM 根据资料组织回答。
```

这两步可以用不同工具：

- LlamaIndex 做检索。
- DeepSeek 做生成。

### API Key 问题不等于 RAG 问题

今天的 401 报错只说明 DeepSeek 认证失败，不代表 LlamaIndex 检索失败。判断问题时要看错误发生在哪一层。

### 当前 embedding 不是最终方案

今天仍使用本地关键词 embedding。它适合学习和验证链路，但不是最终项目方案。后续仍需要选择真实 embedding 模型。
