## 今日核心理解

### 1. LlamaIndex 负责什么

LlamaIndex 是 RAG 项目里的数据连接和检索框架。它可以帮助完成：

- 加载本地文档。
- 把文档变成可检索节点。
- 建立索引。
- 根据问题检索相关节点。
- 把检索结果交给 LLM 生成回答。

### 2. 今日跑通的组件

```text
SimpleDirectoryReader：加载本地文档
VectorStoreIndex：建立向量索引
as_retriever：创建检索器
retrieve：检索相关节点
as_query_engine：创建查询引擎
query：执行检索 + 生成
```

### 3. MockLLM 的作用

今天一开始使用了 `MockLLM`，所以输出是：

```text
text text text ...
```

这不是模型真正回答，只是占位输出。它的作用是帮助我们不依赖真实 API，也能先跑通 LlamaIndex 的框架链路。

### 4. MockEmbedding 的问题

`MockEmbedding` 也只是测试占位，不能判断真实语义检索质量。

当问题是：

```text
SQL 适合查询什么类型的数据？
```

使用 MockEmbedding 时，检索分数可能无法准确体现 SQL 文档更相关。

### 5. 文档粒度影响检索质量

最初所有内容放在一个大文档里，检索结果太粗：

```text
python_learning_notes.md
```

后来拆成多个小文档：

```text
day08_api.md
day09_embedding.md
day10_dify.md
day11_sql.md
```

拆分后，每个文档节点主题更单一，LlamaIndex 可以更细粒度地检索，所以问题更容易命中对应主题。

关键理解：

```text
RAG 检索质量不只取决于模型，也取决于文档切分粒度和 embedding 质量。
```
