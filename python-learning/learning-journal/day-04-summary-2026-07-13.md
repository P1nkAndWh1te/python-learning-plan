### 文件读写

```python
# 读文件
with open('路径', 'r') as f:
    content = f.read()           # 一次性读全部
    line = f.readline()          # 读一行
    lines = f.readlines()        # 读全部，返回列表

# 遍历每一行（推荐）
with open('路径', 'r') as f:
    for line in f:
        print(line)

# 写文件
with open('路径', 'w') as f:
    f.write('内容\n')
```

### 文件模式

| 模式 | 含义 | 注意 |
|:--|--|--|
| `'r'` | 只读 | 文件不存在会报错 |
| `'w'` | 写入 | 会覆盖已有内容 |
| `'a'` | 追加 | 在末尾添加，不覆盖 |
| `'rb'` | 二进制读 | 图片、PDF 等非文本文件 |

### `with` 语句 vs 手动 `close()`

```python
# 手动 — 容易忘 close()
f = open('test.txt', 'r')
content = f.read()
f.close()

# with — 自动关闭，即使中途出错也会关
with open('test.txt', 'r') as f:
    content = f.read()
```

`with` 的好处：代码块结束后自动关闭文件，不用手动 `close()`，即使中间抛出异常也会安全关闭。

### 异常处理

```python
try:
    with open('不存在的文件.txt', 'r') as f:
        print(f.read())
except FileNotFoundError:
    print("文件不存在")
finally:
    print("END")  # 不管有没有异常都会执行
```

- `try`：尝试执行可能出错的代码
- `except 异常类型`：捕获特定异常
- `except` 后面可以不写具体类型，但不推荐（会吃掉所有异常，掩盖 bug）
- `except ... as e`：需要用到异常对象属性时保留，用不到可省略
- `finally`：无论如何都会执行，常用于清理资源

### `split()` 方法

```python
# 无参：按空格切
"hello world".split()         # ['hello', 'world']

# 有参：按指定字符切
"张三,20,北京".split(',')     # ['张三', '20', '北京']
```

### Windows 路径转义问题

```python
# ❌ \t 被当成制表符，\P 是无效转义
open('E:\PythonProject\test.txt', 'r')

# ✅ 三种解法
open(r'E:\PythonProject\test.txt', 'r')   # 加 r 前缀
open('E:/PythonProject/test.txt', 'r')    # 反斜杠改正斜杠
open('E:\\PythonProject\\test.txt', 'r')  # 反斜杠写两遍
```



## 踩过的坑

| 坑 | 原因 | 解法 |
|----|------|------|
| `\P` 和 `\t` 转义错误 | Windows 路径反斜杠被当成转义字符 | 加 `r` 前缀或改用正斜杠 |
| `with (...)` 缺少 `open` | `('path', 'r')` 是元组，不是文件对象 | 写 `with open('path', 'r')` |
| `line = for line in f:` | `for` 本身已经是循环，不能赋值给变量 | `for line in f:` 直接写 |
| `with open(...)` 缺 `as f` | 忘了给文件对象起名 | 写 `with open(...) as f:` |
| `split` 和切片 `L[0:5]` 不是同一个东西 | 切片按位置，split 按分隔符 | 区分记忆 |
