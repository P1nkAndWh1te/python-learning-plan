## 今日知识点

### 1. pip 包管理

| 命令 | 作用 |
|------|------|
| `pip install 包名` | 安装包 |
| `pip uninstall 包名` | 卸载包 |
| `pip list` | 列出当前环境已安装的所有包 |
| `pip freeze > requirements.txt` | 把依赖清单导出到文件 |

**要点**：pip 命令在**命令行（终端/CMD）**执行，不是在 Python 代码里。

---

### 2. 虚拟环境（venv）

**为什么需要？**
- 不同项目依赖不同版本的包，全局安装会冲突
- 虚拟环境给每个项目一个独立的环境，包互不干扰

**基本操作**：

```bash
# 1. cd 到项目目录
cd 你的项目路径

# 2. 创建虚拟环境（文件夹叫 venv）
python -m venv venv

# 3. 激活（Windows CMD）
venv\Scripts\activate.bat

# 3. 激活（Windows PowerShell）
venv\Scripts\Activate.ps1

# 3. 激活（macOS/Linux）
source venv/bin/activate

# 激活成功后，提示符前面会出现 (venv)

# 4. 安装包
pip install requests

# 5. 退出
deactivate
```

### 3. requests 库：发送 HTTP 请求

#### GET 请求（获取数据）

```python
import requests

# 最简单的 GET
response = requests.get("https://api.github.com")

# 带参数（推荐写法，自动编码）
response = requests.get(
    "https://api.github.com/search/repositories",
    params={"q": "search-word", "sort": "stars"}
)
```

#### POST 请求（发送数据）

```python
response = requests.post(
    "https://example.com/api",
    json={"key": "value"}  # 自动转为 JSON
)
```

#### 常用响应属性

| 属性 | 作用 |
|------|------|
| `response.status_code` | HTTP 状态码，200=成功，404=未找到 |
| `response.text` | 返回的原始文本（HTML/JSON 字符串） |
| `response.json()` | 把 JSON 解析成 Python 字典/列表 |
| `response.headers` | 响应头信息 |

#### 请求头（User-Agent）

```python
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}
response = requests.get(url, headers=headers)
```

- `User-Agent` 告诉服务器你是什么客户端
- requests 默认是 `python-requests/版本号`，很多网站会拒绝，所以需要伪装成浏览器
- 格式不固定，直接从浏览器复制一份就能用

### ✅ 练习 1：GitHub API 搜索 "machine learning"

修正了两个常见错误：
- URL 不要用反引号包裹（Markdown 混进代码）
- 请求头名称是 `User-Agent`（连字符），不是 `User_Agent`（下划线）

完整代码：

```python
import requests

headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}
url = "https://api.github.com/search/repositories"

response = requests.get(
    url,
    params={"q": "machine learning", "sort": "stars", "per_page": 5},
    headers=headers
)

data = response.json()
for repo in data["items"]:
    print(f"{repo['name']}  ⭐ {repo['stargazers_count']}")
    print(f"  {repo['html_url']}\n")
```

输出示例：
```
awesome-machine-learning  ⭐ 68502
  https://github.com/josephmisiti/awesome-machine-learning

scikit-learn  ⭐ 56891
  https://github.com/scikit-learn/scikit-learn

tensorflow  ⭐ 179823
  https://github.com/tensorflow/tensorflow

pytorch  ⭐ 79123
  https://github.com/pytorch/pytorch
```

### ✅ 练习 2：天气函数 get_weather(city)

完整代码：

```python
import requests

def get_weather(city):
    url = f"http://wttr.in/{city}?format=3"
    headers = {
        "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
    }
    response = requests.get(url, headers=headers)
    print(response.text)

get_weather("Beijing")
get_weather("Tokyo")
```

学到的技巧：
- f-string 拼接 URL 比 params 更直观
- API 文档在哪里找：官方 README → 网站自带帮助 → 搜索引擎

### ✅ 练习 3：导出 requirements.txt

执行命令：
```bash
pip freeze > requirements.txt
```

生成内容包含：
```
requests==2.34.2
urllib3==2.7.0
...
```

作用：其他开发者拿到你的项目，只需要 `pip install -r requirements.txt` 就能一键安装所有依赖。

## 今日常见错误总结

| 错误 | 原因 | 修复方法 |
|------|------|----------|
| URL 包含反引号 `` ` `` | Markdown 语法直接复制进代码 | 去掉反引号，只用纯字符串 |
| `User_Agent` 下划线 | 拼写错误，记混了 | 改为 `User-Agent` 连字符 |
| headers 定义了但没传进去 | 写完就忘了检查 | `requests.get(url, headers=headers)` |
| 创建 venv 前没 cd 到项目 | 不理解路径概念 | 虚拟环境文件夹会建在当前目录，一定要先进项目目录 |
