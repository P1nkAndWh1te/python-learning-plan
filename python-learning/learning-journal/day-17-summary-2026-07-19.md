## 已实现功能

- 使用固定字符数切分文档。
- 默认 chunk size 为 500 个字符。
- 默认 chunk overlap 为 100 个字符。
- 页面显示 chunk 总数。
- 页面显示每个 chunk 的编号、长度和内容。
- 保留原来的文档预览、问题输入和占位回答。

## 当前边界

今天的切分方式是最小可用版本，按字符长度切分，不按标题、段落或语义切分。

当前应用已经完成：

```text
文档上传 -> 文档读取 -> 文档预览 -> 文档切分 -> chunk 展示
```

还没有完成：

```text
embedding -> Chroma 入库 -> 检索 -> LLM 生成
```

---

## 今日理解

RAG 不能简单地把整篇文档都塞给模型。

文档太长会带来三个问题：

- 超过模型上下文限制。
- 检索时无法精确定位答案片段。
- 回答时容易混入无关内容。

chunk size 控制每段多大，chunk overlap 用来保留相邻片段之间的上下文，避免一句话或一个知识点刚好被切断。
