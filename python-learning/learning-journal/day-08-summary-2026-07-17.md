## 今日知识点

### 1. LLM

LLM 是大语言模型，可以接收文本输入，生成文本输出。后续 RAG 项目中，LLM 负责根据检索到的文档片段组织最终回答。

### 2. Prompt

Prompt 是给模型的输入指令。今天的 prompt 是：

```text
用三句话解释：什么是 API？请用零基础能听懂的中文回答。
```

### 3. API

API 是程序之间约定好的调用接口。一个程序不需要知道另一个程序内部怎么实现，只需要按规定格式发送请求，就能获得数据或服务。

今天模型给出的解释很直观：

- API 像餐厅服务员，负责把你的需求传给厨房，再把结果拿回来。
- API 是两个程序之间约定好的“暗号”。
- 没有 API，每个 App 都要自己造很多功能，会复杂且危险。

### 4. API Key

API Key 是调用服务的凭证，类似密码。

关键规则：

- 不能写进 `.py` 文件。
- 不能发给别人。
- 不能提交到 GitHub。
- 应该从环境变量读取。

### 5. 环境变量

今天使用的环境变量是：

```text
DEEPSEEK_API_KEY
```

Python 读取方式：

```python
import os

api_key = os.environ.get("DEEPSEEK_API_KEY")
```


文件：

- `README.md`

- `deepseek_first_call.py`

核心调用方式：

```python
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com",
)

response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[
        {"role": "system", "content": "你是一个擅长给零基础学生讲编程概念的老师。"},
        {"role": "user", "content": "用三句话解释：什么是 API？请用零基础能听懂的中文回答。"},
    ],
)

print(response.choices[0].message.content)
```
