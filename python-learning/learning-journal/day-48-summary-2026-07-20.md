# Day 48 学习总结：QA Retrieval Mode

今天完成了 DocuAsk v2 的第十一步升级：让 `/qa` 支持检索模式切换。

## 今天真正理解的点

一个成熟一点的 RAG 系统不应该只有一种检索方式。

现在 `/qa` 可以显式选择：

```text
vector：向量检索
bm25：关键词检索
rrf：混合排序
```

默认仍然是 vector，避免破坏旧行为。

## 今天的项目变化

`POST /qa` 新增：

```text
retrieval_mode
```

返回体也会告诉调用方当前使用了哪种模式。

## 今天的验收结果

```text
vector -> Chunk 6
bm25 -> Chunk 6
rrf -> Chunk 6
unknown -> 400
```

## 下一步

下一步给 `/evaluation` 增加 retrieval mode。

这样就能用同一套 10 题评估比较不同检索策略的效果。
