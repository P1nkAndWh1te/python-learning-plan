# Day 29 学习总结：中文 Embedding 依赖评估

## 今天完成了什么

今天没有直接安装中文 embedding 模型，而是先评估了依赖成本。

检查结果：

- 当前 Python 是 3.13.2。
- 当前没有安装 `torch`。
- 当前没有安装 `sentence-transformers`。
- 安装 `sentence-transformers` 会同时带入 `torch`、`transformers`、`tokenizers` 等依赖。

## 为什么不直接安装

因为这是一个较大的依赖变化。

RAG 项目里，embedding 模型不是随便换的。它会影响：

- 安装体积。
- 环境兼容性。
- 模型下载。
- 检索效果。
- 后续部署难度。

所以正确做法是先评估，再确认，再独立实验，最后才考虑接入主应用。

## 今天的关键理解

真实 embedding 有三层成本：

```text
Python 包依赖
模型文件
实际检索效果
```

只要其中一层不合适，就不能直接上线。
