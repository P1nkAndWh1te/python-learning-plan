## 今日知识点

### 1. select

选择要查询的列。

```sql
select name, score
from students;
```

含义：从 `students` 表中查询学生姓名和分数。

### 2. where

过滤符合条件的行。

```sql
select name, city, score
from students
where score >= 90;
```

含义：只查询分数大于等于 90 的学生。

### 3. order by

排序。

```sql
select name, city, score
from students
order by score desc;
```

含义：按分数从高到低排列。

### 4. group by

按某一列分组，然后对每一组做统计。

```sql
select city, avg(score) as avg_score
from students
group by city;
```

理解：

```text
先按 city 分组，再分别计算每个城市学生的平均分。
```

如果查询每个城市有多少学生：

```sql
select city, count(*) as student_count
from students
group by city;
```

含义：在 `students` 表里根据城市分组，查询城市和城市学生数量。

### 5. join

通过两张表之间共同的字段，把信息拼起来。

```sql
select students.name, courses.course_name, students.score
from students
join courses on students.course_id = courses.id;
```

理解：

```text
students.course_id 对应 courses.id。
通过这个关系，可以查出每个学生学的是哪门课程。
```

## 今日重点突破：join + group by

今天新增并跑通：

```sql
select courses.course_name, avg(students.score) as avg_score
from students
join courses on students.course_id = courses.id
group by courses.course_name;
```

理解：

```text
先将 courses 和 students 表通过 course_id / id 连接起来，
再根据课程名分组，
最后查询每门课程和这门课程下学生的平均分。
```

输出：

```text
('Python', 85.5)
('RAG', 89.5)
('SQL', 83.0)
```

---

## 今日核心区分

```text
group by：一张表内部，按某列分组统计。
join：多张表之间，按关联字段拼接信息。
join + group by：先拼表，再按拼出来的信息分组统计。
```
