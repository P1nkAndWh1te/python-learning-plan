# Day 40 学习总结：Retrieval Service

今天完成了 DocuAsk v2 的第三步模块化升级：抽离 retrieval service。

## 今天真正理解的点

RAG 的 retrieval 不是页面功能，而是一段可以复用的后端能力：

```text
chunks -> embedding -> Chroma collection -> query embedding -> top-k chunks
```

Streamlit 只是展示入口。

真正应该沉淀的是：

- 如何把 chunk 写进向量数据库。
- 如何把问题转换为向量。
- 如何从 Chroma 中检索最相关的片段。
- 如何把检索片段组装成后续 LLM 可用的上下文。

## 今天的项目变化

新增：

```text
backend/services/retrieval.py
```

页面 `app.py` 的职责进一步变轻：

- 上传文件。
- 展示 chunk。
- 展示评估结果。
- 展示回答和来源。

检索实现交给 service。

```text
RAG 的基本流程是什么？ -> Chunk 6

```
