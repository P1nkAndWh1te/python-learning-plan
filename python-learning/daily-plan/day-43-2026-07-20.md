# Day 43：新增 QA Endpoint

## 今日目标

继续 DocuAsk v2 升级，新增 FastAPI `POST /qa`，让后端可以根据已写入 Chroma 的文档执行检索。

## 背景

Day42 已经新增：

```text
POST /documents
```

它可以把文档文本切分、向量化并写入 Chroma。

今天继续新增：

```text
POST /qa
```

先只返回检索结果，不调用 LLM 生成最终答案。

## 今日任务

1. 在 retrieval service 中新增按 collection 查询的函数。
2. 在 FastAPI 后端新增 `POST /qa`。
3. 请求体包含 collection name、问题、embedding mode、top_k。
4. 响应体返回 retrieved chunks 和拼接后的 context。
5. collection 不存在时返回 404。
6. 未知 embedding mode 返回 400。
7. 新增独立接口验证脚本。

## 今日边界

- 不调用 DeepSeek。
- 不生成最终自然语言答案。
- 不做多轮对话。
- 不做权限和用户隔离。
- 不改 Streamlit 页面。

## 今日验收标准

今天算完成，当且仅当：

- `POST /documents` 能写入 FAQ 文档。
- `POST /qa` 能检索同一个 collection。
- 问题 `RAG 的基本流程是什么？` Top-1 命中 Chunk 6。
- 缺失 collection 返回 404。
- 原 10 题指标不变。
- GitHub 上有 Day43 提交。
