# Day 42：新增 Documents Endpoint

## 今日目标

继续 DocuAsk v2 升级，让 FastAPI 后端可以通过接口接收文档文本，并写入 Chroma。

## 背景

Day38 到 Day41 已经完成：

- `chunking.py`：文档切分。
- `embeddings.py`：关键词 embedding 和 BGE embedding。
- `retrieval.py`：Chroma 写入、检索、上下文组装。
- `PersistentClient`：本地持久化 Chroma 存储。

今天开始把这些后端能力暴露成 API。

## 今日任务

1. 在 FastAPI 后端新增 `POST /documents`。
2. 定义请求模型：文档文本、embedding mode、chunk size、overlap。
3. 调用 chunking service 切分文档。
4. 调用 retrieval service 写入 Chroma。
5. 返回 document id、collection name、chunk 数量和已存储数量。
6. 对空文档、非法 chunk 参数、未知 embedding mode 返回 400。
7. 新增独立接口验证脚本。

## 今日边界

- 不做 multipart 文件上传。
- 不做 QA endpoint。
- 不调用 DeepSeek。
- 不做多用户权限。
- 不改 Streamlit 页面。

## 今日验收标准

今天算完成，当且仅当：

- `POST /documents` 返回 200。
- FAQ 文档写入后返回 `chunk_count=6`。
- Chroma 中 `stored_chunk_count=6`。
- 未知 embedding mode 返回 400。
- FastAPI `/health` 仍然正常。
- GitHub 上有 Day42 提交。
