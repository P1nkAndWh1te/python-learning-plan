## 今日操作步骤

### 1. 打开 Dify

优先使用 Dify Cloud：

```text
https://cloud.dify.ai/
```

如果你已经有本地 Dify，也可以用本地版本。今天不强求本地部署，因为本日目标是理解 RAG 产品链路，不是学习 Docker。

### 2. 创建知识库

在 Dify 中：

```text
Knowledge -> Create -> Create a ready-to-use knowledge base
```

上传：

```text
02_资料与素材/day10_dify_knowledge/python_learning_faq.md
```

处理策略：

- 先使用默认 chunk 设置。
- 上传后观察 Dify 如何把文档切成 chunks。
- 记录 chunk 数量和你能看到的片段预览。

### 3. 创建 Chatflow 应用

在 Dify 中：

```text
Studio -> Create App -> Chatflow
```

建议应用名：

```text
day10-python-learning-rag
```

### 4. 连接知识库

在 Chatflow 画布中搭建流程：

```text
Start / User Input -> Knowledge Retrieval -> LLM -> Answer
```

Knowledge Retrieval 节点：

```text
Query Text：选择用户输入变量，例如 sys.query / query / user input
Knowledge：选择刚才创建的知识库
```

LLM 节点：

```text
把用户问题和 Knowledge Retrieval 的输出一起放进 Prompt。
要求模型优先根据知识库回答；如果检索结果没有相关内容，就回答“知识库中没有找到相关信息”。
```

Answer 节点：

```text
输出 LLM 节点的回答。
```

### 5. 测试问题

至少问 3 个问题：

```text
1. Day8 学了什么？
2. 为什么 API Key 不能写进代码？
3. Day9 里 embedding 和向量数据库分别是什么？
```

记录：

- 回答是否正确。
- 是否引用了知识库。
- 引用片段是否相关。
- 有没有胡说或答非所问。
