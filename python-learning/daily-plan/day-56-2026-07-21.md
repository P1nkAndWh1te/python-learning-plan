# Day56：DocuAsk 面试追问准备

## 今日目标

把 Day55 的简历最终草稿转成可回答的面试材料，确保每个项目亮点都有解释、证据和边界。

## 验收标准

- 新增一份面试追问材料，覆盖项目介绍、核心技术追问、评测指标、项目不足和后续优化。
- 每个关键说法都能对应到代码、文档、测试或截图。
- 不新增未完成能力，不把小样本评测包装成大规模 benchmark。
- 运行现有测试，确认文档整理没有破坏项目。

## 学习重点

1. 用 60 秒讲清楚 DocuAsk 是什么。
2. 解释 RAG 的端到端链路。
3. 解释 embedding、Chroma、BM25、RRF 的分工。
4. 解释 Top-1 hit 和 Top-k recall。
5. 说明为什么测试里不直接调用真实 LLM API。
6. 主动说明项目边界和下一步优化方向。
