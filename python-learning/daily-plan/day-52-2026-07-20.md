# Day 52：FastAPI Multipart 文件上传

## 今日目标

让 FastAPI 后端支持直接上传 `.txt` / `.md` 文件，而不是只能通过 JSON `text` 字段入库。

## 今日方案

新增接口：

```text
POST /documents/upload
```

保留现有：

```text
POST /documents
```

原因：

- `/documents` 继续服务脚本和测试中的纯文本入库。
- `/documents/upload` 面向真实文件上传场景。
- 两个入口复用同一个入库 helper，避免逻辑分叉。

## 今日任务

1. 新增 multipart 文件上传 endpoint。
2. 支持 `.txt` 和 `.md`。
3. 拒绝其他后缀。
4. 支持 `utf-8` 和 `gbk` 解码。
5. 补充 `python-multipart` 依赖记录。
6. pytest 覆盖成功上传和非法后缀。
7. 新增 Day52 验证脚本和文档记录。

## 今日边界

- 不支持 PDF。
- 不做多文件上传。
- 不做文件大小限制。
- 不改变现有 `/documents` 行为。

## 今日验收标准

- `/documents/upload` 上传 `.md` 返回 200。
- 上传 `.pdf` 返回 400。
- `python -m pytest -q` 通过。
- Day52 验证脚本通过。
