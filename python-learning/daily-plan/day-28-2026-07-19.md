# Day 28：真实 Embedding 最小验证

## 今日目标

用独立脚本验证 Chroma 默认 embedding function 是否能替换教学版关键词 embedding。

验收标准：

- 能读取同一份 `python_learning_faq.md`。
- 能复用当前 Markdown 标题切分逻辑。
- 能用 Chroma 默认 embedding function 建库和检索。
- 能跑同一组 10 个评测问题。
- 能和教学版关键词 embedding 的 70% baseline 对比。

## 今日边界

- 不改 Streamlit 主应用。
- 不引入新依赖。
- 不使用云端 embedding API。
- 不把 DeepSeek API Key 写进代码。

## 学习重点

1. 教学版 embedding 和真实 embedding 的区别。
2. 为什么真实 embedding 能跑通，不代表效果一定更好。
3. 为什么中文 RAG 项目需要选择合适的中文 embedding 模型。
4. 如何用固定评测题判断检索质量，而不是只看单次回答。

## 练习路径

1. 运行 Chroma 默认 embedding 的最小探测。
2. 新增独立评测脚本。
3. 运行 10 题评测。
4. 记录命中率和失败原因。
5. 决定是否进入中文 embedding 模型路线。
