# Day 51：FastAPI 接入 LLM Answer

## 今日目标

让 FastAPI 后端从“只返回 retrieved context”升级为“可以返回 final answer + sources”。

## 今日方案

新增独立接口：

```text
POST /answer
```

而不是修改 `/qa` 的默认行为。

原因：

- `/qa` 继续稳定表示检索和上下文组装。
- `/answer` 表示检索后调用 LLM 生成回答。
- 自动化测试可以继续只稳定验证本地检索链路，不强依赖外部 API。

## 今日任务

1. 新增 `backend/services/generation.py`。
2. 将 DeepSeek prompt 和生成逻辑从 Streamlit 页面抽成可复用 service。
3. 新增 FastAPI `POST /answer`。
4. `/answer` 复用 `retrieval_mode=vector/bm25/rrf`。
5. 无 `DEEPSEEK_API_KEY` 时返回 503。
6. pytest 覆盖 `/answer` 的无 key 和未知 retrieval mode 分支。
7. 新增 Day51 独立验证脚本。

## 今日边界

- 不新增依赖。
- 不把 API Key 写进代码。
- 不让 pytest 调真实 DeepSeek API。
- 不改 `/qa` 的响应语义。

## 今日验收标准

- `/answer` 可接收 `collection_name`、`question`、`embedding_mode`、`top_k`、`retrieval_mode`。
- 无 `DEEPSEEK_API_KEY` 时返回 503。
- 未知 retrieval mode 返回 400。
- `python -m pytest -q` 通过。
- Day51 验证脚本通过。
