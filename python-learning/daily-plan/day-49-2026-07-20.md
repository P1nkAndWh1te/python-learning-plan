# Day 49：Evaluation 支持 Retrieval Mode

## 今日目标

继续 DocuAsk v2 升级，给 FastAPI `POST /evaluation` 增加显式 `retrieval_mode` 参数，用同一套 10 题比较 `vector`、`bm25`、`rrf`。

## 背景

Day48 已经让 `/qa` 支持：

```text
vector
bm25
rrf
```

今天把同样的能力接入 `/evaluation`。

## 今日任务

1. 给 `EvaluationRequest` 增加 `retrieval_mode`。
2. 给 `EvaluationResponse` 增加 `retrieval_mode`。
3. evaluation service 根据 mode 分派到 vector、BM25、RRF。
4. 评估行增加 retrieval mode。
5. 将 `best_distance` 泛化为 `best_score`。
6. 新增 pytest 覆盖三种 mode 和未知 mode。
7. 新增独立验证脚本输出对比表。

## 今日边界

- 不改 Streamlit 页面默认行为。
- 不接 DeepSeek。
- 不调参。
- 不声称 RRF 一定优于 vector，只记录当前数据。

## 今日验收标准

今天算完成，当且仅当：

- `/evaluation retrieval_mode=vector` 可用。
- `/evaluation retrieval_mode=bm25` 可用。
- `/evaluation retrieval_mode=rrf` 可用。
- 未知 mode 返回 400。
- `python -m pytest -q` 通过。
- 原 Day33 指标不变。
- GitHub 上有 Day49 提交。
