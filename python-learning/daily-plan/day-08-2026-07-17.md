## 今日学习顺序

### 1. 概念先行

先记住这几个词：

- **LLM**：大语言模型，输入文本，输出文本或结构化结果。
- **Prompt**：你给模型的输入指令。
- **Token**：模型处理文本的基本计费和长度单位，可以粗略理解为“文字片段”。
- **API**：让你的程序调用远程服务的接口。
- **API Key**：你的调用凭证，类似密码，不能写进代码，不能上传 GitHub。
- **OpenAI-compatible API**：接口格式兼容 OpenAI SDK 的模型服务。代码写法相近，但 `base_url`、环境变量名和模型名可能不同。

### 2. 安全原则

绝对不要这样写：

```python
client = OpenAI(api_key="真实密钥不要这样写进代码")
```

正确做法：

```python
client = OpenAI(
    api_key=os.environ.get("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com",
)
```

今天使用 OpenAI SDK 调用 DeepSeek API。密钥从环境变量 `DEEPSEEK_API_KEY` 读取，不写进代码。

### 4. 设置环境变量

临时设置，只在当前 PowerShell 窗口有效：

```powershell
$env:DEEPSEEK_API_KEY="你的_deepseek_api_key"
```

长期设置，后续新窗口也可用：

```powershell
setx DEEPSEEK_API_KEY "你的_deepseek_api_key"
```

设置后重新打开 PowerShell，再检查：

```powershell
python "03_执行过程/day08_api_basics/env_check.py"
```

## 常见坑

| 坑 | 原因 | 处理 |
|---|---|---|
| `ModuleNotFoundError: No module named 'openai'` | 没安装 SDK | `python -m pip install openai` |
| `DEEPSEEK_API_KEY is not set` | 没设置环境变量 | 用 `$env:DEEPSEEK_API_KEY=...` 或 `setx` |
| 设置了 `setx` 但脚本还读不到 | 当前终端没刷新 | 关闭 PowerShell，重新打开 |
| 把 API Key 写进 `.py` 文件 | 安全风险 | 立即删除，重新生成 Key，确认 Git 没提交 |
| 把 `.env` 提交到 GitHub | 泄露密钥 | `.gitignore` 必须包含 `.env` |
| 仍使用 `deepseek-chat` | 旧模型名即将弃用 | 使用 `deepseek-v4-flash` 或 `deepseek-v4-pro` |
