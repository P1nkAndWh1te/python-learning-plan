# Day 37：FastAPI 后端服务骨架

## 今日目标

进入 DocuAsk v2 Step 1，新增 FastAPI 后端服务骨架。

## 背景

当前 RAG QA System 已经是可运行的 Streamlit 本地原型，但后端服务边界还不清晰。

v2 的第一步不是迁移全部 RAG 逻辑，而是先建立可独立运行的后端服务：

```text
GET /health
```

## 今日任务

1. 确认当前环境中 `fastapi` 和 `uvicorn` 可用。
2. 将 `fastapi` 和 `uvicorn` 写入 `requirements.txt`。
3. 新增 `backend/` 包。
4. 新增 FastAPI app。
5. 新增 `/health` 接口。
6. 新增后端 README。
7. 本地启动并验证 `/health`。

## 今日边界

- 不迁移 RAG 核心逻辑。
- 不改 Streamlit 页面。
- 不新增持久化 Chroma。
- 不新增文档上传接口。
- 不新增 pytest。

## 今日验收标准

今天算完成，当且仅当：

- `python -m py_compile backend/app.py` 通过。
- `python -m uvicorn backend.app:app --app-dir "04_成果输出/rag-qa-system" --host 127.0.0.1 --port 8000` 能启动。
- `GET /health` 返回 `status: ok`。
- 现有 Streamlit 应用不受影响。
- GitHub 上有 Day37 提交。
