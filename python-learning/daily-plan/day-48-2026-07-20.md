# Day 48：QA 支持 Retrieval Mode

## 今日目标

继续 DocuAsk v2 升级，给 FastAPI `POST /qa` 增加显式 `retrieval_mode` 参数，支持 `vector`、`bm25`、`rrf`。

## 背景

Day46 完成了 BM25 baseline。

Day47 完成了 RRF baseline。

今天把它们接入 `/qa`，但默认行为仍保持 `vector`。

## 今日任务

1. 给 `QaRequest` 增加 `retrieval_mode`。
2. 默认值保持 `vector`。
3. `/qa` 根据 mode 分派到 vector、BM25 或 RRF。
4. 返回体增加 `retrieval_mode`。
5. retrieved chunk 支持 `distance`、`score`、`rrf_score`。
6. 未知 mode 返回 400。
7. 新增 pytest 和独立验证脚本。

## 今日边界

- 不改 `/evaluation`。
- 不改 Streamlit 页面。
- 不接 DeepSeek。
- 不改变 `/qa` 默认行为。

## 今日验收标准

今天算完成，当且仅当：

- `/qa` 默认 `vector` 仍可用。
- `/qa retrieval_mode=bm25` 可用。
- `/qa retrieval_mode=rrf` 可用。
- 三种模式对 `RAG 的基本流程是什么？` 都 Top-1 命中 Chunk 6。
- 未知 mode 返回 400。
- `python -m pytest -q` 通过。
- GitHub 上有 Day48 提交。
