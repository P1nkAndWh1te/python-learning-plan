## 今日核心概念

### 1. Embedding

Embedding 就是把文本、图片或其他数据变成一串数字。

例如：

```text
"Python 调用 API" -> [1.0, 0.8, 0.0, 0.2]
"Git 提交代码"   -> [0.0, 0.1, 1.0, 0.0]
```

这些数字不是给人看的，而是给计算机比较相似度用的。

### 2. 向量相似度

如果两个文本的向量方向接近，就说明它们语义更接近。今天先用余弦相似度理解：

```text
相似度越接近 1：越相似
相似度越接近 0：越不相关
```

### 3. 向量数据库

普通数据库擅长：

```sql
where city = 'Beijing'
```

向量数据库擅长：

```text
找出和“怎么调用接口”语义最相似的几段文本
```

RAG 里的向量数据库主要负责：

1. 存储文档片段。
2. 存储每个片段对应的 embedding。
3. 根据用户问题的 embedding 找最相似的片段。
4. 把这些片段交给 LLM 生成回答。
