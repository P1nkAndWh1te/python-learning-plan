## 学习顺序

### 1. 先理解 Git 的四个位置

不用一开始背很多命令，先记住这条线：

```text
工作区 -> 暂存区 -> 本地仓库 -> 远程仓库
修改文件    git add    git commit    git push
```

对应含义：

- **工作区**：你正在编辑的文件夹。
- **暂存区**：下一次提交准备包含哪些改动。
- **本地仓库**：你电脑里的提交历史。
- **远程仓库**：GitHub 上的备份和同步位置。

今天必须搞懂：`git add` 不是保存代码，`git commit` 才是把一次改动写进历史记录。

---

### 2. 本地仓库最小闭环

新建一个练习目录，例如：

```powershell
mkdir python-learning-record
cd python-learning-record
git init
```

创建 `hello.py`：

```python
print("Hello Git")
```

执行：

```powershell
git status
git add hello.py
git status
git commit -m "Add hello git script"
git log --oneline
```

检查点：

- `git status` 能看懂哪些文件未跟踪、哪些文件已暂存。
- `git log --oneline` 能看到刚才那次提交。

### 3. 整理 Day 1-6 学习代码

建议结构：

```text
python-learning-record/
  README.md
  day01_basics.py
  day02_functions_modules.py
  day03_collections.py
  day04_files_exceptions.py
  day05_requests_demo.py
  day06_json_pandas.py
  requirements.txt
  .gitignore
```

如果你之前的代码分散在不同位置，今天不需要重构，只做最小整理：

- 每天保留 1 个代表性 `.py` 文件。
- 代码能不能全部跑通不是第一优先，先让目录结构清楚。
- 不要把 `venv/`、`__pycache__/`、临时 CSV/JSON 输出文件一股脑提交进去。

建议 commit 节奏：

```powershell
git add README.md
git commit -m "Add learning record readme"

git add day01_basics.py day02_functions_modules.py day03_collections.py day04_files_exceptions.py
git commit -m "Add day 1 to day 4 python exercises"

git add day05_requests_demo.py day06_json_pandas.py requirements.txt .gitignore
git commit -m "Add package and data processing exercises"
```

---

### 4. `.gitignore` 必须写

创建 `.gitignore`：

```gitignore
venv/
__pycache__/
*.pyc
.env
*.log

# 练习代码运行时生成的临时文件，可按需保留或忽略
data.json
json_file.json
students.csv
```

注意：如果某个文件已经被 Git 跟踪，后来再写进 `.gitignore` 不会自动取消跟踪。这一点今天了解即可。

### 5. GitHub 推送（可选但建议做）

本机已经有 Git，可以直接用命令行提交。本机没有检测到 `gh` 命令，所以 GitHub 新仓库建议你在网页上创建。

网页创建空仓库后，复制 GitHub 给你的 remote 命令，通常类似：

```powershell
git remote add origin https://github.com/你的用户名/python-learning-record.git
git branch -M main
git push -u origin main
```

如果推送时要求登录，按浏览器或 Git Credential Manager 的提示完成即可。

## 常见坑预警

| 坑 | 原因 | 处理 |
|---|---|---|
| `git commit` 提示没有用户名邮箱 | Git 不知道提交作者是谁 | `git config --global user.name "你的名字"` 和 `git config --global user.email "你的邮箱"` |
| `git status` 里出现一堆 venv 文件 | 没写 `.gitignore` 或写晚了 | 先写 `.gitignore`，再 `git status` 检查 |
| `git add .` 把无关文件也加进去了 | 当前目录太大或位置不对 | 先 `pwd` / `ls` 确认目录，再 add |
| push 失败 | 远程仓库地址、登录、权限问题 | 先保证本地 commit 成功，再单独处理 GitHub |
| commit message 写成 `update` | 信息太模糊 | 用动词开头，如 `Add day 6 json exercise` |
