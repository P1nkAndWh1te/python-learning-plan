# Day14 自测题

请用自己的话回答。

## 1. RAG 是什么？

回答：

```text
RAG 是检索增强生成。它让 AI 在回答前先从指定知识库或文档中检索相关资料，再根据这些资料生成回答。这样可以减少模型凭空编造的概率，让回答更有依据，但不能 100% 消除幻觉。
```

## 2. RAG 为什么不是让 LLM 直接回答？

回答：

```text
因为 LLM 直接回答时只能依赖模型已有知识，而这些知识可能过时、不完整，或者不包含我们的私有文档。如果它不知道答案，还可能编出看起来像真的内容。RAG 先检索指定资料，再让 LLM 基于资料回答，可以让答案更准确，也更容易追溯来源。
```

## 3. Embedding 模型和 LLM 有什么区别？

回答：

```text
Embedding 模型负责把文本转换成向量，用来做语义相似度检索；LLM 负责根据输入的 prompt 生成自然语言回答。在 RAG 里，检索系统先找到相关文档片段，再把这些片段交给 LLM，让它基于资料组织答案。
```

## 4. 向量数据库负责什么？

回答：

```text
向量数据库负责存储文档片段和它们对应的 embedding 向量。用户提问时，系统也会把问题转成向量，然后向量数据库根据相似度找出最接近的问题相关文档片段。
```

## 5. LlamaIndex 在项目里负责什么？

回答：

```text
LlamaIndex 是 RAG 框架，负责组织 RAG 检索链路。它可以加载文档、切分文档、调用 embedding 模型生成向量、建立索引、检索相关片段，并把检索结果交给 LLM。Embedding 是把文本转成向量的模型或方法，LlamaIndex 是把文档、embedding、索引、检索和 LLM 串起来的框架。
```

## 6. 为什么文档切分粒度会影响检索质量？

回答：

```text
文档切分粒度会影响检索质量。粒度太大时，一个 chunk 里可能包含多个主题，检索命中后会带入很多无关信息，影响 LLM 判断。粒度太小时，一个完整语义可能被切断，LLM 拿到的上下文不完整，也会影响回答质量。所以需要在 chunk_size 和 chunk_overlap 之间做平衡。
```

## 7. SQL 和向量检索有什么区别？

回答：

```text
SQL 主要用于查询结构化数据，比如表格里的行和列，可以用 where、group by、join 精确筛选和统计。向量检索主要用于查询非结构化文本，比如 Markdown、PDF、Word 文档里的自然语言内容，它先把文本转成向量，再根据语义相似度找相关片段。
```

## 8. 当前项目为什么用 DeepSeek 做生成，但 embedding 后续单独选择？

回答：

```text
因为当前使用的 DeepSeek 主要适合做聊天/推理生成，项目里已经验证它可以作为 OpenAI-compatible LLM API 生成回答。但 embedding 是另一类能力，需要把文本转换成向量；当前 DeepSeek 不是我们已确定可用的 embedding 方案，所以 embedding 后续要单独选择，比如本地 BGE、Jina Embeddings 或其他 embedding API。
```
