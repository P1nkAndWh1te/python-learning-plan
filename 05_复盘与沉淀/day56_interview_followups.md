# Day56 复盘：从项目完成到面试可解释

## 复盘结论

简历项目不能只停留在“完成了功能”，还要能解释每个亮点背后的工程判断。

DocuAsk 当前可以稳定讲的内容：

- RAG 端到端链路。
- FastAPI 后端接口拆分。
- Chroma 本地持久化。
- vector / BM25 / RRF 检索对比。
- Top-1 hit / Top-k recall 小样本评测。
- pytest 覆盖核心接口和异常分支。

## 风险点

- RRF 90% / 100% 只能说是当前小样本结果。
- 不能把本地原型包装成生产级系统。
- 不支持 PDF / Word、rerank、多用户权限、压测和线上监控。

## 面试策略

遇到追问时，不急着堆技术栈，先说清楚问题，再说实现和验证。

推荐回答顺序：

```text
背景 -> 问题 -> 行动 -> 结果 -> 边界
```

这个结构能避免两类问题：

- 只说“我用了某某技术”，但没有说明技术解决了什么问题。
- 夸大项目能力，被追问后无法提供证据。
